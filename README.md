## خط الإنتاج (Excel → ETL → مرقوم)

### 1. المصدر: ملفات Excel
كل كشاف مبني من ملف Excel أصلي يحتوي على أوراق تحليلية.  
الملفات الأصلية محفوظة في S3 ولا يمكن الوصول إليها مباشرة.

### 2. ETL: توليد view_cache.json
```bash
cd /home/ubuntu/marqoom
python3 etl/marqoom_etl.py path/to/kashaf.xlsx \
  --book-id marqoom-XX \
  --title "اسم الكتاب" \
  --author "اسم المؤلف" \
  --category "حديث" \
  --died-hijri 852 \
  --output-dir etl/output/kashaf_id/
```
يولّد الأمر:
- `marqoom_XX_view_cache.json` — بيانات العرض السريع
- `marqoom_XX_validation_report.json` — تقرير التحقق

### 3. التسجيل في قاعدة البيانات
```bash
node etl/register_viewcache.mjs kashaf_id etl/output/kashaf_id/marqoom_XX_view_cache.json
```
يرفع الملف إلى S3 ويسجّل المعرّف في جدول `kashaf_viewcache`.

### 4. API الداخلي
```
GET /api/trpc/kashafat.getViewCache?input={"json":{"kashafId":"fathalbaari"}}
```
يجلب البيانات من S3 عبر قاعدة البيانات — لا يظهر أي .json في Network tab.

## مزامنة المعرّفات
لمزامنة معرّفات قاعدة البيانات مع معرّفات الواجهة:
```bash
node etl/sync_ids.mjs
```
الخريطة موثّقة في `etl/sync_ids.mjs` مع تعليق لكل كشاف.

## الهيكل التقني

```
client/src/pages/
  Home.tsx          ← الصفحة الرئيسية (70 كشاف)
  KashafDetail.tsx  ← صفحة تفصيلية لكل كشاف
  
server/routers.ts   ← tRPC procedures (kashafat.getViewCache)
drizzle/schema.ts   ← جدول kashaf_viewcache

etl/
  marqoom_etl.py         ← سكريبت ETL الرسمي
  register_viewcache.mjs ← تسجيل في قاعدة البيانات
  sync_ids.mjs           ← مزامنة المعرّفات
  output/                ← مخرجات ETL
```
