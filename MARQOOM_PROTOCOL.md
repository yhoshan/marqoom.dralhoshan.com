# MARQOOM_PROTOCOL.md
# بروتوكول مرقوم لملفات Excel — النسخة 1.0.0

> **الغرض:** هذا الملف هو المرجع الرسمي لبنية ملفات Excel في مشروع **مرقوم — بوابة الكشافات التراثية الرقمية**.
> كل ملف Excel ملتزم بهذا البروتوكول يمكن تحويله آلياً إلى `view_cache.json` بواسطة ETL مرقوم دون الحاجة إلى تعديل الكود.

---

## 1. مبادئ البروتوكول

- **Excel هو المصدر الوحيد للبيانات (Single Source of Truth)**
- **JSON هو View Cache للعرض فقط** — يُولَّد آلياً من Excel ولا يُعدَّل يدوياً
- **Word يُولَّد من قاعدة البيانات عند الطلب** — ليس مصدراً مستقلاً
- **الأوراق الأساسية (Core)** يجب أن تكون موجودة في كل ملف
- **الأوراق الاختيارية (Modules)** تُضاف حسب طبيعة الكتاب
- **غياب ورقة اختيارية لا يوقف التحويل** — يُسجَّل في `validation_report.json`
- **غياب ورقة أساسية يُسجَّل كخطأ** في `validation_report.json` مع استمرار التحويل

## 4. بنية الأعمدة في الأوراق الأساسية

### C1 — لوحة عامة
الأعمدة المتوقعة (بأي ترتيب):
- `المؤشر` أو `البند` — اسم المؤشر
- `القيمة` — القيمة الرقمية أو النصية
- `ملاحظة` أو `أساس العد/التنبيه` — تنبيه اختياري

**المؤشرات الرئيسية المستخرجة:**
- عدد الصفحات
- عدد الأجزاء
- عدد الكلمات التقريبي
- عدد الحروف
- عدد السور (للتفاسير)

### C2 — بيانات الملفات
الأعمدة المتوقعة:
- `اسم الملف` — اسم ملف Excel
- `الصيغة` — نوع الملف
- `الحجم بالبايت` أو `الحجم`
- `عدد الصفحات الداخلية` أو `الصفحات`
- `عدد الكلمات`
- `عدد الحروف`

### C3 — توزيع الأغراض
الأعمدة المتوقعة:
- `الغرض` أو `العبارة/المصطلح` أو `الفئة` — اسم الغرض
- `العدد` — عدد الإحالات
- `النسبة%` أو `النسبة` — النسبة المئوية

### C4 — الموارد حسب المجال
الأعمدة المتوقعة:
- `المجال` — اسم المجال
- `العدد` أو `مؤشر الحضور`
- `النسبة` — النسبة المئوية

### C5 — حدود الدراسة
الأعمدة المتوقعة:
- `الحد/التنبيه` أو `البند` — وصف الحد
- `الأثر على القراءة` أو `البيان` — التفسير

### C6 — المؤشرات المركبة
الأعمدة المتوقعة:
- `المؤشر` — اسم المؤشر
- `العدد` أو `البسط/القيمة 1`
- `المقام الحسابي` أو `المقام/القيمة 2`
- `النسبة` أو `الناتج/النسبة`
- `التنبيه` أو `أساس الحساب`

## 5. قواعد ETL

### 5.1 تطبيع أسماء الأوراق
يستخدم ETL مطابقة مرنة (Fuzzy Matching) لتحديد الورقة الصحيحة:
1. مطابقة تامة أولاً
2. مطابقة جزئية (الاسم الأساسي موجود في اسم الورقة)
3. مطابقة الأسماء البديلة المدرجة في الجدول أعلاه

### 5.2 تطبيع أسماء الأعمدة
يستخدم ETL مطابقة مرنة للأعمدة:
- إزالة المسافات الزائدة
- تجاهل الفرق بين `%` و`نسبة` و`النسبة%`
- تجاهل الأرقام في بداية اسم العمود

### 5.3 معالجة القيم المفقودة
- ورقة أساسية غائبة → `null` في JSON + تحذير في Validation Report
- ورقة اختيارية غائبة → حقل محذوف من JSON + ملاحظة في Validation Report
- عمود غائب في ورقة موجودة → `null` للحقل + تحذير

### 5.4 أنواع البيانات
- الأرقام: دائماً `int` أو `float` — لا نصوص رقمية
- النسب المئوية: `float` بين 0 و100
- الأرقام في JSON: دائماً بالإنجليزية (0-9)

## 9. قواعد التسمية

- `book_id`: `marqoom-{رقم}` — مثال: `marqoom-12`
- `schema_version`: `1.0.0`
- اسم ملف الناتج: `marqoom_{رقم}_view_cache.json`
- اسم ملف التقرير: `marqoom_{رقم}_validation_report.json`
